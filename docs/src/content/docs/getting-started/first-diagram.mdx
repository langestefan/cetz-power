## Anatomy of a diagram

Every snippet on this site has the same two-line scaffold around the
actual symbol calls:

```typst
#import "@preview/cetz-power:0.1.0": *

#diagram({
  // your symbols go here
})
```

The `#diagram({ ... })` wrapper is **required** — it isn't decoration.
Skipping it (calling raw `cetz.canvas` for example) means the symbols
won't render the way you expect.

### What `diagram` does

`diagram(body, ..params)` is a ~20-line wrapper around CeTZ's own
`cetz.canvas(...)`. It does two things `cetz.canvas` doesn't:

1. **Installs the default style dictionary.** Every symbol in this
   package reads its defaults — strokes, fills, sizes, label
   positions — from the `cetz-power` key in CeTZ's style context.
   `diagram` is what puts the dictionary there. Without it the
   three-layer style cascade (global → family → per-call, see
   [Styling](/getting-started/styling/)) has no base to merge against,
   so global and family-level overrides via `set-style(cetz-power: …)`
   silently do nothing.
2. **Handles HTML output.** When Typst is compiled to HTML (which is
   how the diagrams on this site are produced), `diagram` wraps the
   result in `html.frame(...)` so it renders as inline SVG. CeTZ
   canvases otherwise produce nothing in HTML mode because they
   internally use Typst's `layout()` primitive, which HTML mode
   ignores. PDF compilation is unaffected.

Any extra named arguments (e.g. `length: 1.2cm` to set the unit-to-cm
scale) are forwarded straight to `cetz.canvas`.

### Why the wildcard `: *` import

The `{ ... }` block after the function call is a Typst *code block* —
a fresh lexical scope that inherits names from the surrounding scope.
Importing the package with `: *` puts every symbol (`diagram`, `bus`,
`transformer`, …) directly in the surrounding scope, so the block can
call them without any prefix.

If you imported the package with `as pg` instead — e.g. to avoid
exporting cetz-power's names into your global namespace — you'd need
to either prefix every call (`pg.bus(…)`, `pg.transformer(…)`) or add
`import pg: *` as the first line inside the canvas block to re-export
the symbols locally:

```typst
#import "@preview/cetz-power:0.1.0" as pg

#pg.diagram({
  import pg: *  // re-export symbols into this code block
  bus("b1", (0, 0))
})
```

Both forms compile to the same output. The wildcard form is what every
example on this site uses; the namespaced form is there if you want
the explicit `pg.` prefix.

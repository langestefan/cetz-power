A stroked circle with optional content inside — the conventional
single-line notation for rotating machines, sources, and meters. The
letter that goes inside picks the meaning; common ones:

- **V** — voltage source
- **A** — asynchronous machine
- **G** — generator
- **M** — motor
- **W** — wattmeter

<Snippet name="machine-example" />

## Parameters

Positional form is `machine(name, pos, letter?)`. A named `letter:` arg
(if passed) takes precedence over the positional one.

| Name          | Default  | Description                                            |
| ------------- | -------- | ------------------------------------------------------ |
| `letter`      | `none`   | Content drawn inside the circle — a string like `"V"`, math like `[$Phi$]`, any content. Accepted positionally as the third argument or as a named arg. |
| `radius`      | `0.3`    | Circle radius.                                         |
| `stroke`      | `0.8pt`  | Stroke override.                                       |
| `fill`        | `none`   | Fill colour.                                           |
| `letter-size` | `10pt`   | Font size of the inner content (applied only when `letter` is a string). |
| `label`       | `none`   | External label (standard label dict).                  |

## Anchors

`center`, `north`, `south`, `east`, `west` — the four cardinal edges,
sitting on the circle. Plus `north-east`, `south-east`, `south-west`,
`north-west` — the four 45° diagonals, also on the circle. All eight
are safe as wire endpoints or label-attachment points.

## Labels

External annotations (machine ratings, names, etc.) use the same
`label:` dict as every other cetz-power symbol — set the world-space
`anchor:` to position the label, `distance:` to push it further away:

<Snippet name="machine-labels" />

For a caption tucked into one of the corners, use the diagonal anchors
(`north-east`, `south-east`, `south-west`, `north-west`). The built-in
`label:` dict only attaches one caption at a time; if you want multiple
corner tags on the same machine, draw extra ones with
`cetz.draw.content(<anchor>, anchor: <opposite>, …)`:

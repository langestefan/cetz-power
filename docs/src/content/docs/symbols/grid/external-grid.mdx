Single-terminal connection representing "the grid beyond this point" —
an infinite bus. Drawn as a cross-hatched square.

<Snippet name="grid-example" />

## Parameters

| Name      | Default | Description                                                |
| --------- | ------- | ---------------------------------------------------------- |
| `size`    | `0.9`   | Side length of the square.                                 |
| `lead`    | `0.2`   | Stub from connection point to the bottom of the square.   |
| `fill`    | `none`  | Fill colour of the square.                                 |
| `stroke`  | `0.8pt` | Stroke override.                                           |
| `line-count` | `2` | Hatching density. `0` = bare square (no hatching); `1` = just the two main diagonals (a bare X); `2` adds the conventional cross-hatch; higher values give a progressively denser grid. Must be `>= 0`. |
| `background` | `none` | Fill colour for the inside of the square, drawn under the cross-hatching so the chord lines remain visible on top. Same shape as `fill`; the name reads more naturally for "tint behind the hatching". |

## Hatching density

`n = 0` draws a bare square with no hatching at all. From `n = 1`
onwards the two main corner-to-corner diagonals are added, and each
higher `n` overlays additional evenly-spaced chord pairs to produce a
denser cross-hatch:

<Snippet name="grid-line-count" />

Override per call (`external-grid("g", (0,0), line-count: 4)`) or
globally via `set-style(cetz-power: (grid: ("line-count": 4)))`.

## Anchors

`in` (= connection point, `(0,0)`), `center`, `north` (top of the square),
`default`.

## Rotation

Rotate with `angle:` when you want the symbol to stick out of a
horizontal bus instead of hanging below one:

## Fancier examples

### Voltage-level colour coding

The same red / orange / blue convention used elsewhere in the
diagram for voltage levels works on the grid stroke too. Pair it
with `line-count` to give a visual cue about source strength —
denser hatching reads as a stiffer upstream system:

<Snippet name="grid-voltage-coded" />

### Multi-line labels with engineering data

The `label:` argument accepts arbitrary Typst content, so a
multi-line label can carry the numbers a power-system engineer
actually wants to see — short-circuit power, X/R ratio, nominal
voltage. Combine with a tinted `fill:` to make the source pop:

<Snippet name="grid-rich-label" />

### Tinted background

`background:` fills the inside of the square (under the
cross-hatching) — useful for colour-coding source strength,
voltage level, or system zone without changing the stroke colour.
The hatching stays on top so the symbol still reads as a grid:

<Snippet name="grid-background" />
